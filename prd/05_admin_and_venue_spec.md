# Раздел 5. Админ-панель и Спецификация Venue (Omad Arena v1)

Учитывая, что мы выбрали стэк **React + TypeScript + Vite**, для создания мощного дашборда управления (Backoffice) целесообразно применить фреймворк **Refine.dev**. Он идеально оборачивает REST API (через dataProvider) и позволяет за считанные минуты развернуть CRUD-таблицы.

## 5.1 Конфигурация UI и Ролей

Доступ в этот проект будет только у сущностей с ролью `ADMIN`, `SUPER_ADMIN` или `VENUE`. Обычных `USER` система скидывает на экран Unauthorized.

### Панель для VENUE (Точка выдачи / Кассир)
Так как VENUE нужно сканировать QR коды с камеры смартфона (iPad, терминала), для этой роли доступен крайне ограниченный набор экранов интерфейса:

1. **Главный экран (Сканер):**
   - На весь экран открывается компонент Camera (например, пакет `react-qr-reader` или нативный API браузера `Zxing`).
   - При наведении на QR, отправляется запрос на `POST /api/v1/venue/scan`.
   - Открывается "Шторка подтверждения": "Канал: Кофешоп X. Приз: Капучино. Выдать?".
   - Кнопка **[Выдать]** дергает `/api/v1/venue/redeem` c `Idempotency-Key` (защита от двойной выдачи).
2. **Экран Истории (Redemptions Log):**
   - Простая таблица со списком призов, которые этот кассир/заведение выдало за сегодня/месяц.

### Панель для ADMIN / SUPER_ADMIN
У администраторов полная панель навигации (Sidebar):

1. **Dashboard (Дашборды фаундеров):**
   - Карточки с Big Numbers: MAU, DAU, Суммарно начислено Points, Суммарно куплено призов.
   - Интеграция с Recharts (графики по дням).

2. **Меню "Users" (Пользователи):**
   - Таблица (ID, Телефон, Username, Balance, Status).
   - Кнопка *Block/Ban* (вызывает окно "Точно забанить?").
   - Редактирование роли (только для SUPER_ADMIN).

3. **Меню "Games & Limits" (Настройки Игр):**
   - Список 9 игр.
   - Поле: `Score To Points Ratio` (Например: 100/1).
   - Поле: `Max Score Limit` (например: 3000 в минуту).
   - Выпадающий список: Привязка игры к конкретной Сети Заведений (`Venue Network`).

4. **Меню "Venue Networks" (Сети спонсоров):**
   - Добавление новых сетей. (Например, создание "Brand Area Cola").

5. **Меню "Prizes Shop" (Магазин Призов):**
   - Создание приза. Указание цены в Points, загрузка картинки в Object Storage (S3-compatible) через pre-signed URL.
   - Выбор Сети/Игры, к которой привязан приз.
   - Переключатель "В наличии/Изъят".

6. **Меню "Promo Codes" (Промокоды):**
   - Генерация новых промо. 
   - Выставление Dates (Range picker) и лимита активаций.
   - Красная кнопка экстренного отключения (Deactivate Force).

## 5.2 Интеграция с Telegram Bot (Support)

Для модуля технической поддержки мы не будем сразу разрабатывать громоздкий in-house чат.

**Флоу работы Support:**
1. В приложении пользователь заходит в раздел "Помощь" и пишет текст сообщения.
2. Бэкенд NestJS ловит этот текст, создает в БД `SupportTicket`.
3. Далее Бэкенд делает банальный запрос в Telegram Bot API. Бот присылает сообщение в рабочую группу разработчиков/саппорта в Telegram: 
   *"Новый тикет от +99890xxxx! Проблема с игрой Тетрис."*
4. На этапе v1 менеджер отвечает прямо из Telegram (используя Reply ID), Бот подхватывает ответ и отправляет его обратно в NestJS, который перенаправляет юзеру PUSH-уведомлением.
